# 结构化规划指南

这些文件用于维护长篇小说写作前的结构层。JSON 文件便于 CLI 和 Codex 读取，Markdown 文件便于人工说明和审阅。

## 文件用途

- `volumes.json`：卷级规划，例如 `第一卷` 的目标、核心冲突、主要人物、关键伏笔和章节范围。
- `chapter_plans.json`：章级规划，例如 `第001章` 的目标、视角、场景、出场人物、伏笔推进和结尾钩子。
- `timeline.json`：时间线事件，记录事件顺序、章节号、时间标签、示例地点、影响和备注。
- `arcs.json`：人物线、关系线、事业线或主题线，例如 `主角A` 与 `角色B` 的关系推进。
- `scenes.json`：场景卡，记录场景位置、人物、目的、冲突、结果和关联伏笔。
- `conflicts.json`：冲突线，记录示例冲突的引入、升级、解决、利害关系和状态。

## 使用原则

- 规划文件是写作前的结构层，不要把完整正文写入规划 JSON。
- JSON 用于机器可读结构，Markdown 用于人工阅读说明。
- 示例内容只使用通用占位，例如 `主角A`、`角色B`、`示例地点`、`示例伏笔`、`第一卷`、`第001章`。
- 用户真实作品内容应保存在独立小说项目中，不要提交到工具仓库本体。

## 推荐流程

1. 用 `plan-init` 初始化规划文件。
2. 用 `add-volume` 添加卷纲。
3. 用 `add-chapter-plan` 添加章纲。
4. 用 `add-scene` 添加场景卡。
5. 用 `add-timeline-event` 添加时间线事件。
6. 用 `add-arc` 添加人物线或关系线。
7. 用 `planning-status` 检查规划完整度。
8. 用 `outline-export` 导出人工可读大纲。
