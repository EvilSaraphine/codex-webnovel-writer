## Folder Roles

- `设定集/`: stable canon such as world rules, factions, locations, power systems, and style constraints.
- `设定集/AI句式禁区.md`: editable anti-AI phrasing and style risk rule notes used by `style-rules` and `style-check`.
- `大纲/`: total outline, volume outlines, chapter outlines, timelines, and arc plans.
- `正文/`: chapter drafts and revisions.
- `审查报告/`: review output for consistency, pacing, AI flavor, foreshadowing, character state, continuity checks, and doctor reports.
- `审查报告/doctor-report.md`: read-only project health report generated by `doctor`.
- `审查报告/第XXX章-deep-review.md`: structured single-chapter review generated by `review`.
- `审查报告/第XXX章-style-report.md`: read-only AI phrasing and style risk report generated by `style-check`.
- `审查报告/style-summary-第XXX-第YYY章.md`: range summary generated by `style-check-range`.
- `审查报告/review-summary-第XXX-第YYY章.md`: range summary generated by `review-range`.
- `templates/设定集/AI句式禁区.md`: starter rule note file copied into initialized projects.
- `templates/审查报告/doctor-report-template.md`: template used for doctor reports in initialized projects.
- `templates/审查报告/deep-review-template.md`: template used for single-chapter deep review reports.
- `templates/审查报告/style-report-template.md`: template used for style-check reports.
- `templates/审查报告/review-summary-template.md`: template used for chapter range review summaries.
- `伏笔记录/`: setup and payoff tracking.
- `人物状态/`: current character knowledge, goals, relationships, wounds, resources, and secrets.
- `章节索引/`: progress table for outline, draft, review, publication readiness, summaries, and generated context packs.
- `章节索引/context-packs/`: chapter-specific writing context packs generated by local retrieval.
- `章节索引/write-briefs/`: chapter-specific writing task briefs.
- `章节索引/write-briefs/第XXX章-write-brief.md`: writing task brief generated by `write-brief`.
- `templates/章节索引/write-brief-template.md`: template used for writing task briefs.
- `章节索引/write-workspace/`: chapter writing workspace instructions.
- `章节索引/write-workspace/第XXX章-write-instruction.md`: instruction file generated by `write`.
- `templates/章节索引/write-workspace/write-instruction-template.md`: template used for write instruction files.
- `templates/正文/draft-template.md`: draft template used by `write` when the chapter draft does not exist.
- `.webnovel/retrieval-config.json`: local retrieval configuration.
- `.webnovel/chunks.json`: generated local retrieval chunks.
- `.webnovel/retrieval-index.json`: generated local retrieval index metadata.

## Doctor Reports

`doctor` checks project structure, JSON readability, chapter index consistency, summaries, structured planning, character state, hook records, local retrieval index status, and generic content safety. It writes `审查报告/doctor-report.md` and does not automatically repair project files.

## Deep Review Reports

`review` checks one chapter using local files and writes `审查报告/第XXX章-deep-review.md`. `review-range` runs the same checks over a chapter range and writes `审查报告/review-summary-第XXX-第YYY章.md`. These reports are diagnostic and do not modify drafts or project state.

## Style Reports

`style-check` checks one chapter for AI-flavored phrasing and style risk, then writes `审查报告/第XXX章-style-report.md`. `style-check-range` runs the same heuristic checks over a chapter range and writes `审查报告/style-summary-第XXX-第YYY章.md`. These reports are read-only and require human confirmation before any prose changes.

## Write Briefs

`write-brief` generates `章节索引/write-briefs/第XXX章-write-brief.md` from chapter plans, character state, hooks, timeline events, scene cards, conflicts, context packs, review reports, and summaries. `prepare-write` can refresh index, retrieval index, context pack, and write brief, but it does not generate chapter prose.

## Write Workspace

`write` prepares `章节索引/write-workspace/第XXX章-write-instruction.md`, a chapter draft, context pack, and write brief. `finalize-write` runs summary, index, local retrieval rebuild, deep review, and doctor after正文 is written. Neither command calls external services or automatically updates character, hook, outline, or setting files.

## Context Policy

Do not load the whole project for every task. Pick files by workflow:

- Planning: settings, total outline, nearby outlines, foreshadowing, character state.
- Writing: target chapter outline, nearby drafts, relevant canon and state.
- Review: target draft, source outline, nearby continuity files.
- Query: search the relevant record folder first, then inspect outlines or drafts if needed.
